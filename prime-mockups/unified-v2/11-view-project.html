Real-time Object Detection on ESP32-S3
Deploy a custom TensorFlow Lite model for real-time object detection using the ESP32-S3's PSRAM and camera interface — no cloud required.
JH
Jakob Huber
OpenMV Published Feb 14, 2026
•
48 forks
Introduction

The ESP32-S3 is a dual-core Xtensa LX7 microcontroller with 512KB of SRAM and support for up to 8MB of octal-PSRAM — more than enough headroom to run quantized TensorFlow Lite models at useful inference speeds. Combined with its built-in camera interface (DVP), the S3 makes an ideal candidate for edge computer vision projects where you want self-contained, battery-powered detection without relying on a cloud backend.

This project walks you through training a lightweight MobileNetV2-based object detection model using TensorFlow Model Maker, quantizing it to INT8, and deploying it on an ESP32-S3 DevKit board connected to an OV2640 camera module. The final system achieves approximately 10 FPS with bounding box overlay rendered on an attached SPI LCD.

By the end, you'll have a fully working detection pipeline that can recognize up to 10 object classes from a custom training dataset — entirely on-device, with a total BOM cost under $65.

Setting Up the Development Environment

We'll use the ESP-IDF v5.2 framework with the esp-tflite-micro component, which provides a pre-compiled TF Lite Micro runtime optimized for Xtensa LX7. Start by installing ESP-IDF and cloning the project template.

BASH terminal
$ git clone --recursive https://github.com/espressif/esp-idf.git ~/esp/esp-idf $ cd ~/esp/esp-idf && git checkout v5.2.1 $ ./install.sh esp32s3 Installing tools for ESP32-S3... xtensa-esp32s3-elf: downloaded $ source export.sh $ git clone https://github.com/snaplify-examples/esp32s3-tflite-detect $ cd esp32s3-tflite-detect && idf.py set-target esp32s3

Next, install Python dependencies for the training pipeline and model conversion step. A virtual environment is strongly recommended to avoid version conflicts with other projects.

BASH requirements.txt
$ python3 -m venv .venv && source .venv/bin/activate $ pip install tflite-model-maker==0.4.2 tensorflow==2.12 $ pip install pillow numpy opencv-python-headless Successfully installed tflite-model-maker-0.4.2

2
Flashing the Firmware

With the environment configured, build the firmware and flash it to the ESP32-S3 over USB. The board uses a USB-Serial-JTAG interface built into the SoC — no external USB-UART bridge needed.

BASH
$ idf.py build Project build complete. Binary size: 1.42 MB esp32s3-tflite-detect.bin $ idf.py -p /dev/tty.usbmodem* flash monitor Chip is ESP32-S3 (revision v0.2) Features: WiFi, BLE, Embedded Flash 16MB, Embedded PSRAM 8MB Flashing... (100%) I (432) app_main: Camera init OK — 320x240 JPEG I (488) tflite: Model loaded, 892 KB, INT8 quantized I (489) tflite: Starting inference loop @ 10 FPS target
Tip: If the flash command fails with Permission denied, add your user to the dialout group on Linux (sudo usermod -aG dialout $USER) or use sudo on macOS. On Windows, install the CP210x driver if the device doesn't enumerate.
Warning: Make sure to hold the BOOT button while pressing EN/RST to enter download mode on first flash if your board doesn't automatically enter DFU. Some DevKitC-1 batches have the auto-download circuit omitted.
3
Training the Custom Model

TensorFlow Model Maker makes it straightforward to fine-tune a pre-trained MobileNetV2 feature extractor on your own dataset. Prepare your images in Pascal VOC XML format or COCO JSON — Model Maker handles both. For this project, we trained a 10-class detector on ~800 annotated images per class.

PYTHON train.py
import tflite_model_maker as mm from tflite_model_maker import object_detector from tflite_model_maker.config import ExportFormat # Load dataset — supports COCO and Pascal VOC format train_data = object_detector.DataLoader.from_pascal_voc( images_dir='dataset/train/images', annotations_dir='dataset/train/annotations', label_map={1: 'person', 2: 'electronics', 3: 'tools'} ) val_data = object_detector.DataLoader.from_pascal_voc( images_dir='dataset/val/images', annotations_dir='dataset/val/annotations', label_map={1: 'person', 2: 'electronics', 3: 'tools'} ) # Use EfficientDet-Lite0 — optimized for microcontrollers spec = object_detector.EfficientDetLite0Spec() model = object_detector.create( train_data, model_spec=spec, validation_data=val_data, epochs=50, batch_size=8, train_whole_model=True ) # Export quantized INT8 TFLite model model.export( export_dir='./output', tflite_filename='detect_int8.tflite', export_format=[ExportFormat.TFLITE], quantization_config=mm.config.QuantizationConfig.for_int8(representative_data=train_data) ) print(f"Model size: {os.path.getsize('output/detect_int8.tflite') / 1024:.1f} KB")

Training on a GPU-equipped machine (NVIDIA RTX 3060 was used here) takes roughly 40–60 minutes for 50 epochs. The resulting INT8-quantized model file is approximately 890 KB — well within the 8MB PSRAM available for model storage and inference buffers.

Results & Testing

After flashing, point the camera at objects in your training set. The LCD overlay renders bounding boxes at approximately 10 frames per second with the inference pipeline running on core 1 while core 0 handles camera capture and display refresh.
