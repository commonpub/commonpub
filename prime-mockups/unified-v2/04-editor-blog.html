I've been obsessed with local AI for the past year. Not the kind that phones home to a server farm in Virginia, but the kind that runs quietly in your closet, on your own hardware, answering your voice without sending a single packet to the cloud. Last month I finally got it working — a fully offline voice assistant running on a Raspberry Pi 4 (4GB) with a custom wake-word model trained on my own voice.

Why Run It Locally?

The commercial offerings — Alexa, Google Home, Siri — are convenient. But every utterance you make travels to a data center, gets transcribed, analyzed, and filed. If you've ever read the fine print, you know they retain voice samples. My home lab runs Home Assistant, Jellyfin, and a self-hosted LLM via Ollama. Adding a cloud-dependent voice layer felt like a contradiction.

The Hardware Stack

  • Raspberry Pi 4 (4GB) — enough headroom to run Whisper tiny comfortably
  • ReSpeaker 4-Mic Array — far-field beamforming, way better than the Pi's onboard nothing
  • 32GB SanDisk Extreme A2 microSD — wear-leveled, faster random read for model loading
  • PoE HAT — single cable to the Pi hidden behind the TV
  • Pimoroni Speaker pHAT — small 3W speaker for TTS output

The wake-word engine is openWakeWord, an open-source library that runs on CPU and produces surprisingly low false-positive rates once you train a custom model. The training pipeline requires about 500 positive samples of your chosen wake phrase. I went with "hey atlas" — short, distinct phonemes, nothing that appears naturally in conversation.

python
import openwakeword from openwakeword.model import Model import pyaudio import numpy as np # Load custom model trained on "hey atlas" oww_model = Model( wakeword_models=["./models/hey_atlas.tflite"], inference_framework="tflite" ) p = pyaudio.PyAudio() stream = p.open(format=pyaudio.paInt16, channels=1, rate=16000, input=True, frames_per_buffer=1280) while True: audio = np.frombuffer(stream.read(1280), dtype=np.int16) prediction = oww_model.predict(audio) if prediction["hey_atlas"] > 0.5: print("Wake word detected! Listening...") transcribe_and_respond()
Tip
The frames_per_buffer=1280 value is important — openWakeWord expects 80ms audio chunks at 16kHz. Mismatching this causes silent failures where the model runs but never triggers. Also make sure your ReSpeaker is set to mono output before the beamforming stage.

STT: Whisper on the Edge

Once the wake word fires, I have about a 2-second window to start recording the command. The audio gets piped to faster-whisper running the tiny.en model — 39M parameters, quantized to int8. On the Pi 4, a 5-second utterance transcribes in about 1.2 seconds. Not blazing fast, but acceptable for a home automation context where you're already used to a half-beat delay.

"The goal isn't to beat cloud latency — it's to be good enough while keeping your data yours." — inspiration from the local-LLM community on r/selfhosted

The full project — wiring diagrams, systemd service files, training scripts, and pre-trained weights for "hey atlas" — is published on my profile. If you build your own variant, drop a photo in the comments. I want to see what wake words people choose.
